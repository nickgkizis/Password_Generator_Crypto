Pseudo-Random vs. Cryptographically Secure Random Numbers

When generating passwords, it is important to use randomness that is secure and unpredictable. Below is an explanation of the two types of randomness used:

  • Pseudo-random Numbers: These are generated by algorithms like Math.random() in JavaScript. While they appear random, they are deterministic, meaning if you know the starting point (or seed), the sequence will be predictable.
  • Cryptographically Secure Random Numbers: These are generated by system-level entropy sources (e.g., using crypto.getRandomValues() in JavaScript). These numbers are not predictable and are ideal for secure applications like password generation.

This app uses cryptographically secure random numbers to generate passwords, making them much more secure and harder to predict.
